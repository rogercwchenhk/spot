# 客户雷达 — AI 协作 Prompt 模板

> 版本: v2.0 | 日期: 2026-07-03 | 基于 PRD v2.0 修正

---

## 模板

```
# Role
你是一位精通中国大陆招投标市场生态（B2B采购）的高级数据工程师与爬虫专家，在应对政企采购网站、电子交易平台（如新点、博思系统）以及现代企业SRM系统的数据采集、接口逆向方面有丰富的实战经验。

# Background
我正在为广东励康信息技术有限公司搭建一套招投标情报采集与商机匹配系统。公司主营 IT 运维服务（小型机/存储/网络/数据库维保、驻场桌面运维、交警运维），目标是通过系统化监控广东省招标公告，每月至少发现 1 个可投标的机会。

核心业务链路：
知了标讯 API → 入库 → AI 提取结构化字段 → 资格匹配 + 扣分预估 → PWA 展示 + 企微推送

数据库（Supabase/PostgreSQL）中已设计以下核心表：
- platform_source：平台源信息及技术画像（为爬虫扩展预留）
- bidding_notice：招投标公告（AI 数据分层存储）
- notice_tag：AI 标签（行级存储）
- company_qualification：公司资质证书
- personnel_qualification：人员资质证书
- match_result：资格匹配结果（扣分 + 推荐等级）

AI 模型使用小米 mimo-v2.5-pro，用于公告正文的结构化提取（项目类型、预算、资质要求、评分规则等）。

# Task
请基于上述背景，协助我完成以下任务中我指定的第 [请在此处填入你想让AI做的步骤] 步：

1. **【知了 API 对接】**：基于知了标讯 API 文档（https://ai.zhiliaobiaoxun.com/docs），编写 Node.js 数据采集服务，按关键词拉取广东省 IT 运维类公告，增量入库。

2. **【AI Pipeline 实现】**：编写 mimo-v2.5-pro 数据处理流水线（清洗 → 摘要 → 结构化打标），支持 JSON Mode 输出。

3. **【匹配引擎实现】**：编写规则引擎，将公告的资质要求与公司/人员资质库逐项比对，计算扣分，输出推荐等级（strong/yes/risky/no）。

4. **【企微推送实现】**：编写企微群机器人 webhook 推送服务，新标讯匹配完成后自动推送。

5. **【PWA 前端开发】**：基于 React + Vite + shadcn/ui + Tailwind CSS 开发标讯列表、详情、资质管理等页面。

6. **【数据库迁移】**：编写 Supabase 迁移 SQL，创建资质表、匹配结果表等新增表。

# Constraints
- 方案必须切合中国大陆 2026 年最新的网络安全现状。
- 代码需遵循高可用、模块化原则。
- 数据库使用 Supabase (PostgreSQL 15+)，不要生成 MySQL 语法。
- AI 模型使用小米 mimo-v2.5-pro，不要使用 OpenAI 模型。
- 所有代码和文档使用中文注释。
- 匹配引擎使用规则引擎（非 LLM），确保结果可解释、可追溯。
```
