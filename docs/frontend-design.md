# 客户雷达 — 前端设计文档

> 版本: v1.0 | 日期: 2026-07-03 | 基于实施计划 v2.0
> 状态: 待审批

---

## 1. 设计原则

- **信息密度优先**：桌面端充分利用屏幕空间，表格/列表密集展示
- **移动端友好**：关键操作在手机上可达，PWA 可安装到主屏幕
- **国际化视觉**：shadcn/ui 质感，克制配色，大量留白，清晰层级
- **响应式断点**：`sm: 640px` / `md: 768px` / `lg: 1024px` / `xl: 1280px`

---

## 2. 认证与权限

### 2.1 角色定义

| 角色 | 能力 | 适用人群 |
|---|---|---|
| admin | 所有页面 + 资质增删改 + 平台管理 + 推送配置 | 管理者 |
| viewer | 标讯列表/详情/搜索（只读）+ 资质查看（只读） | 销售人员 |

### 2.2 认证流程

\`\`\`
用户访问任意页面
      │
      ├─ 已登录 ──→ 正常访问，按角色展示内容
      │
      └─ 未登录 ──→ 跳转登录页
\`\`\`

- 使用 Supabase Auth 内置的邮箱密码登录
- 登录后 Supabase 返回 JWT，前端存储在 localStorage
- API 请求自动携带 Authorization header
- RLS 策略按角色控制数据读写权限

\`\`\`
# 首次登录（交互式输入邮箱密码）
cr login

# 登录后 token 存储在 ~/.cr/auth.json
# 后续命令自动携带 token

# 切换角色（需要对应账号）
cr login --role admin

# 登出
cr logout
\`\`\`
